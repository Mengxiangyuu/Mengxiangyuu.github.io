Review of Federated Learning for Medical Image Analysis(2)

客户端：有限的数据和标签 (Client end: Limited data and labels)

问题背景：有限数据和标签

在医学图像分析的联邦学习（FL）中，许多站点（如医院、诊所等）常常面临数据量不足的挑战，特别是在医学图像中，标注数据的缺乏使得训练有效模型更加困难。由于医学影像标注的成本高昂，客户端可能只有有限的医学图像和标签，这会导致模型在训练时无法充分学习到有用的特征，进而影响全局模型的训练效果。因此，如何处理这些小样本数据的问题，成为联邦学习中的一个重要研究课题。

解决方案

  1. 对比学习 (Contrast Learning)
  2. 多任务学习 (Multi-Task Learning)
  3. 弱监督学习 (Weakly-Supervised Learning)
  4. 知识蒸馏 (Knowledge Distillation)
  5. 数据合成 (Data Synthesis)

联邦学习中的异构环境问题

问题背景：计算资源与数据差异

本研究关注的是在不同的医疗影像站点/中心之间，数据量和计算资源（如GPU的可用性）上的差异。尤其是当计算资源差异较大时，强者更强，弱者更弱，导致整个训练过程的收敛速度变慢。

举例

一个拥有先进GPU的站点进行50个周期的更新可能只需要1秒钟，而计算资源较弱的站点可能需要100秒。这样，强者客户端需要等待较长时间，直到其他较弱客户端完成训练并进行模型共享，从而降低了联邦训练的整体效率。

解决方案：半同步训练

Stripelis等人提出了一种半同步训练策略，专门用于医学影像分析任务。该策略允许不同客户端根据其计算资源和数据量的不同，进行不同的训练周期数。

服务器端学习中的权重聚合

问题背景：权重聚合

在联邦学习系统中，需要有效的策略来聚合客户端的权重，确保每次客户端与服务器通信后，系统性能能够保持一致，避免性能退化。

解决方案：

Progressive Fourier 聚合策略

Chen等人提出了一种在服务器端的Progressive Fourier 聚合策略（基于傅里叶变换）。该策略通过聚合客户端的低频参数部分来共享不同客户端之间学习到的知识，而忽略高频部分。这种方法基于先前的研究，认为深度网络的能力主要由低频部分决定。

训练损失的影响

Li等人考虑了每个客户端的训练损失，作为权重聚合的影响因素。例如，在COVID-19检测中，数据不均衡的客户端可能会表现较差。对于这些表现较差的客户端，其权重会被调整为较小值，从而影响全局权重的聚合。

服务器端学习中的客户端领域漂移问题

问题背景：领域漂移

客户端之间的领域漂移，即不同医疗中心（客户端）之间的图像异质性，可能会导致全局模型偏差。具体来说，一个全局模型可能在某些客户端表现良好，但在其他客户端表现较差。

解决方案：

Hosseini等人的方法

Hosseini等人提出了修正的优化目标（受到无线网络中公平资源分配方法的启发），以促进统一的模型表现。当全局模型在某些客户端上表现较差时，会将其贡献添加到总损失函数中。

Fan等人的方法

Fan等人利用引导梯度来优化全局模型。他们在聚合各客户端的本地权重时，只使用聚合后权重的正值来更新全局模型。

Luo等人的方法

Luo等人提出了一种名为共享标签分布的联邦学习（FedSLD）方法，特别适用于医学图像分类任务。该方法通过缓解客户端之间标签分布的差异，来解决领域漂移问题。

服务器端学习中的客户端异常检测

问题概述：客户端异常检测

本研究探索如何保护联邦学习系统免受来自客户端的影响，尤其是由图像标签噪声或恶意攻击造成的腐败。

解决方案：异常检测

Alkhunaizi等人提出了一种针对医学图像的服务器端异常检测方法，称为基于距离的异常值抑制（DOS）。该方法能够有效防止客户端故障。每个客户端的权重是根据客户端的异常得分计算的，异常得分是通过Copula-based outlier detection方法进行计算的。

客户端-服务器通信中的数据泄露与攻击

问题背景：数据泄露与隐私侵犯

开发有效的方法，防止在联邦学习的客户端与服务器交互过程中，医疗图像数据泄露和隐私侵犯。

解决方案：

部分权重共享

Yang等人提出，分享整个模型（网络）可能无法完全保护隐私，因此他们建议在医学数据集上使用部分模型共享进行联邦学习。

通信效率优化

问题背景：通信效率

制定优化客户端与服务器之间通信的策略，旨在加速联邦学习（FL）中的收敛过程，并确保更有效的模型训练。

解决方案：动态融合方法

Zhang等人提出了一种基于动态融合的FL方法，专门用于COVID-19诊断。该框架根据客户端模型的性能动态选择参与的客户端进行权重融合。

常见数据集使用方式：

  • 将大型数据集拆分成多个子集

未来研究方向

重点关注：

  2. 多模态融合用于联邦学习：如何在联邦学习中利用多模态医学图像数据。

Review of Federated Learning for Medical Image Analysis

Challenges

1. Single Site or Center → Limited Sample Size

  • Small sample size is a significant problem in medical image analysis. Data from a single site is often insufficient for training robust models.

2. Bias in Data Distribution

  • Inherent biases exist in datasets from specific sites or centers. This includes:
    • Unbalanced data (e.g., more healthy subjects than patients)
    • Local features: Characteristics specific to the local patient population.
  • Federated learning can help mitigate this problem by aggregating data from multiple centers without needing to share actual data.

Federated Learning

Client-Server Architecture

  • Client Selection: Clients (sites) are selected, and the server initializes a global model.
  • Model Upload: Clients train the model on local data and send updates (not raw data) to the server.
  • Aggregation: The server aggregates all client model updates to form a new global model, which is then broadcast to all clients.
  • Recent Model Advances: More advanced models, such as the Vision Transformer, are being introduced for client training.
    • 重点关注 1：新的模型: 列举哪些新的模型
  • New Aggregation Algorithms: Recent models explore adaptive weighting strategies to improve model aggregation efficiency.
    • 重点关注 2：新的聚合算法, 列举哪些新的算法
  • Optimizing Communication Protocols: Reducing communication overhead is critical to enhancing federated learning efficiency.
    • 重点关注 3：优化通信协议 (communication protocols) 和最小化通信开销
  • Iteration and Convergence: With each round, the global model becomes more refined and accurate.
    • 重点关注 4：高整体训练效率 (training efficiency) 和加速收敛 (convergence)

Types of Federated Learning

1. Horizontal Federated Learning

  • Different hospitals or medical institutions collaborate while keeping their own patient data.
  • Each hospital’s patient data (e.g., X-ray images) are similar in features but different in samples.

Example:

Three hospitals (A, B, and C) use chest X-ray images to diagnose lung diseases. Each hospital has its own patient data:

  • Hospital A: Collects lung X-ray images, extracting features like lung size, shape, and texture.
  • Hospital B: Has similar patient images but different individuals.
  • Hospital C: Contains lung X-ray data with similar features.

Through horizontal federated learning, these hospitals train a joint diagnostic model without sharing patient data. Each hospital updates its local model and sends it to a central server, which aggregates all updates into a more robust global model.

Is privacy the only advantage of horizontal federated learning?
Yes, one of the main advantages of horizontal federated learning is data privacy protection.

2. Vertical Federated Learning

  • Different institutions hold different types of data for the same group of patients.

Example:

A hospital and a research lab:

  • Hospital: Has patient imaging data (X-ray, CT scans) to detect disease symptoms.
  • Research Lab: Has the same patients’ genomic data (genetic sequencing results), which helps assess genetic risks.

Three Categories of Federated Learning in Medical Image Analysis

  1. Client-End Methods
  2. Server-End Methods
  3. Communication Methods

Client-End Methods

Challenges:

1. Domain Shift

  • Problem: Medical images from different sites may have different distributions due to variations in scanning settings or patient populations.
  • Impact: Causes instability in machine learning models and slows convergence in federated learning.

2. Data Heterogeneity

  • Problem: Different hospitals may use different imaging techniques (X-ray, MRI, CT).
  • Impact: Makes model updates difficult to integrate, reducing overall model effectiveness.

Solutions for Client-End Issues

1. Domain-Specific Learning

  • Goal: Reduce negative impact of client shift by locally fine-tuning the global model.
1.1. Encoding-Decoding Structure (Feng et al. - 2019)
  • Used for MR image reconstruction.
  • What is Encoding-Decoding?
    • The encoding phase extracts relevant features.
    • The decoding phase reconstructs images.
1.2. CNN and GNN (Chakravarty et al. - 2020)
  • Combined Convolutional Neural Networks (CNN) and Graph Neural Networks (GNN).
  • CNN captures site-independent features.
  • Local GNN fine-tunes data for each hospital.
1.3. Ensemble Framework (Xu et al. - 2021)
  • Solution for medical image segmentation with client shift.
  • Consists of:
    • Global Model
    • Personalized Models
    • Model Selector (chooses best model per client)
1.4. Local and Global Gradient Combination (Jiang et al. - 2022)
  • Uses local gradients (from client data) and global gradients (aggregated from all clients).
1.5. GAN-Based Methods for Medical Imaging
  • Ke et al. (2022): Used GANs to harmonize histopathology images.
  • Wagner et al. (2022): Used a reference dataset to train local GANs for image harmonization.

2. Domain Adaptation

  • Addresses differences in medical imaging across hospitals, equipment, and patient populations.

Key Domain Adaptation Methods

  1. Li et al. (2018):

    • Added noise to data for privacy.
    • Used a discriminator to reduce domain shift.

  2. Dinsdale et al. (2019):

    • Modeled medical image features as Gaussian distributions.
    • Shared mean and standard deviation across clients.

  3. Andreu et al. (2020):

    • Used Batch Normalization (BN) to handle domain shift.

  4. Guo et al. (2021):

    • Applied federated learning-based domain adaptation for MRI reconstruction.

3. Image Harmonization

  • Reduces visual inconsistencies in medical images from different clients.

Key Image Harmonization Methods

1. VAE & GAN (Qu et al. - 2020)
  • Used Variational Autoencoder (VAE) to generate synthetic medical images.
2. CycleGAN for Image Harmonization (Yan et al. - 2021)
  • Used CycleGAN to align medical image styles.
  • Selected a low-complexity client as a reference.
3. Frequency Domain-Based Harmonization (Jiang et al. - 2021)
  • Converted medical images into frequency and phase domains.
  • Shared mean amplitude across clients for harmonization.

Conclusion for Client end: Domain shift among client

HTML基础-5

定位

使用position属性来设置定位

可选值：

static 默认值，元素是静止的没有开启定位

relative 开启元素的相对定位

absolute 开启元素的绝对定位

fixed 开启元素的固定定位

sticky 开启元素的粘滞定位

相对定位

当元素的position属性值设置为relative时则开启了元素的相对定位

相对定位的特点：

  1. 元素开启相对定位以后，如果不设置偏移量元素不会发生任何的变化
  2. 相对定位是参照于元素在文档流中的位置进行定位的
  3. 相对定位会提升元素的层级
  4. 相对定位不会改变元素的性质（块还是块，行内还是行内）

偏移量

当元素开启了定位以后，可以通过偏移量来设置元素的位置

属性：top，bottom，right，left

绝对定位

当元素的position属性值设置为absolute时，则开启了元素的绝对定位

绝对定位的特点：

  1. 开启绝对定位后，如果不设置偏移量元素的位置不会发生变化
  2. 开启绝对定位后，元素会从文档流中脱离
  3. 绝对定位会改变元素的性质，行内变成块，块的宽高被内容撑开
  4. 绝对定位会使元素提升一个层级
  5. 绝对定位元素是相对于其包含块进行定位的

包含块

正常情况下：

包含块就是离当前元素最近的祖先块元素

绝对定位的包含块:

包含块就是离它最近的开启了定位的祖先元素，

如果所有的祖先元素都没有开启定位则根元素就是它的包含块

html（根元素、初始包含块）

固定定位

将元素的position属性设置为fixed则开启了元素的固定定位

固定定位也是一种绝对定位，所以固定定位的大部分特点都和绝对定位一样

唯一不同的是固定定位永远参照于浏览器的视口进行定位

粘滞定位

当元素的position属性设置为sticky时则开启了元素的粘滞定位

粘滞定位和相对定位的特点基本一致，

不同的是粘滞定位可以在元素到达某个位置时将其固定

HTML基础-4

浮动简介

通过浮动可以使一个元素向其父元素的左侧或右侧移动

使用 float 属性来设置于元素的浮动

可选值：

none 默认值 ，元素不浮动

left 元素向左浮动

right 元素向右浮动

注意，元素设置浮动以后，水平布局的等式便不需要强制成立

元素设置浮动以后，会完全从文档流中脱离，不再占用文档流的位置，

所以元素下边的还在文档流中的元素会自动向上移动

浮动的特点

  1. 浮动元素不会盖住文字，文字会自动环绕在浮动元素的周围，所以我们可以利用浮动来设置文字环绕图片的效果

  2. 脱离文档流的特点：

    块元素：

     1、块元素不在独占页面的一行

 2、脱离文档流以后，块元素的宽度和高度默认都被内容撑开

    行内元素：

     行内元素脱离文档流以后会变成块元素，特点和块元素一样

    脱离文档流以后，不需要再区分块和行内了

高度塌陷的问题

在浮动布局中，父元素的高度默认是被子元素撑开的，

当子元素浮动后，其会完全脱离文档流，子元素从文档流中脱离

将会无法撑起父元素的高度，导致父元素的高度丢失。

父元素高度丢失以后，其下的元素会自动上移，导致页面的布局混乱。

BFC

Block Formatting Context 块级格式化环境

BFC是一个CSS中的一个隐含的属性，可以为一个元素开启BFC

开启BFC该元素会变成一个独立的布局区域。

元素开启BFC后的特点：

  1. 开启BFC的元素不会被浮动元素所覆盖
  2. 开启BFC的元素子元素和父元素外边距不会重叠
  3. 开启BFC的元素可以包含浮动的子元素

开启BFC

将元素的overflow设置为一个非visible的值

常用的方式 为元素设置 overflow:hidden/auto 开启其BFC 以使其可以包含浮动元素.

HTML基础-3

文档流

网页是一个多层的结构，一层摞着一层

通过CSS可以分别为每一层来设置样式

作为用户来讲只能看到最顶上一层

这些层中，最底下的一层称为文档流，文档流是网页的基础

盒模型

CSS将页面中的所有元素都设置为了一个矩形的盒子

每一个盒子都由一下几个部分组成：

    内容区（content）
    内边距（padding）
    边框（border）
    外边距（margin）

内容区

元素中的所有的子元素和文本内容都在内容区中排列

内容区的大小由width 和 height两个属性来设置

边框

边框属于盒子边缘，边框里边属于盒子内部，除了边框都是盒子的外部

边框的大小会影响到整个盒子的大小

边框的属性：

    边框的宽度 border-width

    边框的颜色 border-color

    边框的样式 border-style

border-width

border-width可以用来指定四个方向的边框的宽度

border-xxx-width

xxx可以是 top right bottom left

用来单独指定某一个边的宽度

border-style

属性：

    solid 表示实线
    dotted 点状虚线
    dashed 虚线
    double 双线

内边距

内容区和边框之间的距离是内边距

内边距的设置会影响到盒子的大小

背景颜色会延伸到内边距上

一共有四个方向的内边距：

    padding-top
    padding-right
    padding-bottom
    padding-left

外边距

外边距不会影响盒子可见框的大小

但是外边距会影响盒子的位置

一共有四个方向的外边距：

    margin-top
    - 上外边距，设置一正值，元素会向下移动
    margin-right
     - 默认情况下设置margin-right不会产生任何效果
    margin-bottom
    - 下外边距，设置一个正值，其下边的元素会向下移动
    margin-left
    - 左外边距，设置一个正值，元素会向右移动

    - margin也可以设置负值，如果是负值则元素会向相反的方向移动

盒子的水平布局

元素在其父元素中水平方向的位置由以下几个属性共同决定

    margin-left
    border-left
    padding-left
    width
    padding-right
    border-right
    margin-right

过度约束

一个元素在其父元素中，水平布局必须要满足以下的等式：

margin-left+border-left+padding-left+width+padding-right+border-right+margin-right = 其父元素内容区的宽度 （必须满足）

如果不满足，就是过度约束

如果某个值为auto，则会自动调整为auto的那个值以使等式成立

所以我们经常利用这个特点来使一个元素在其父元素中水平居中

盒子的垂直布局

默认情况下父元素的高度被内容撑开

如果子元素的大小超过了父元素，则子元素会从父元素中溢出

使用 overflow 属性来设置父元素如何处理溢出的子元素

可选值：

    visible，默认值 子元素会从父元素中溢出，在父元素外部的位置显示

    hidden 溢出内容将会被裁剪不会显示

    scroll 生成两个滚动条，通过滚动条来查看完整的内容

    auto 根据需要生成滚动条

外边距的折叠

相邻的垂直方向外边距会发生重叠现象

会出现以下几种情况：

兄弟元素

  • 兄弟元素间的相邻垂直外边距会取两者之间的较大值（两者都是正值）
  • 特殊情况：
  1. 如果相邻的外边距一正一负，则取两者的和
  2. 如果相邻的外边距都是负值，则取两者中绝对值较大的

父子元素

  • 父子元素间相邻外边距，子元素的会传递给父元素（上外边距）

这个在开发中会带来问题，比如给子元素设置上外边距，让子元素往下移，父元素也会同样往下移。

行内元素的盒模型

行内元素不支持设置宽度和高度

行内元素可以设置padding、border、margin但是垂直方向padding不会影响页面的布局

display 用来设置元素显示的类型

可选值：

    inline 将元素设置为行内元素
    
    block 将元素设置为块元素
    
    inline-block 将元素设置为行内块元素 
    
    行内块，既可以设置宽度和高度又不会独占一行
    
    table 将元素设置为一个表格
    
    none 元素不在页面中显示

visibility 用来设置元素的显示状态

可选值：

    visible 默认值，元素在页面中正常显示

    hidden 元素在页面中隐藏 不显示，但是依然占据页面的位置

盒子的尺寸

box-sizing 用来设置盒子尺寸

可选值：

    content-box 默认值，宽度和高度用来设置内容区的大小
    
    border-box 宽度和高度用来设置整个盒子可见框的大小

轮廓和圆角

box-shadow

用来设置元素的阴影效果，阴影不会影响页面布局

    第一个值 水平偏移量 设置阴影的水平位置 正值向右移动 负值向左移动

    第二个值 垂直偏移量 设置阴影的水平位置 正值向下移动 负值向上移动

    第三个值 阴影的模糊半径

    第四个值 阴影的颜色

用来设置元素的轮廓线，用法和border一模一样

轮廓和边框不同的点，就是轮廓不会影响到可见框的大小

border-radius

用来设置圆角 圆角设置的圆的半径大小

样式的继承

为一个元素设置的样式同时也会应用到它的后代元素上

选择器的权重

样式的冲突：
当我们通过不同的选择器，选中相同的元素，并且为相同的样式设置不同的值时，此时就发生了样式的冲突。

发生样式冲突时，应用哪个样式由选择器的权重（优先级）决定

选择器的权重
内联样式 1,0,0,0

id选择器 0,1,0,0

类和伪类选择器 0,0,1,0

元素选择器 0,0,0,1

通配选择器 0,0,0,0

继承的样式 没有优先级

比较优先级时，需要将所有的选择器的优先级进行相加计算，最后优先级越高，则越优先显示

（分组选择器是单独计算的）

div,p,span{
    background-color:yellowgreen;
}//div,p,span三个分开计算

选择器的累加不会超过其最大的数量级，类选择器在高也不会超过id选择器

如果优先级计算后相同，此时则优先使用靠下的样式
